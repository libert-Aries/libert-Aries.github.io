# java 中的流程控制

循环嵌套：一个循环写在另一个循环里面，外层循环一个数字对应整个内层循环

循环控制：

continue [继续]: 跳过本次循环 开始下一次 跳到所在循环的第三部分 break [中断]: 跳出所在的循环 跳到所在循环的结束部分

continue 和 break 都只能跳出所在的循环

循环标签：当多个循环嵌套在一起的时候 如果代码写在最里面一个循环 想要处理外层循环 需要给 外层循环贴标签

标识符： [a:] 然后在内层循环里面 continue/break a;

# java 中的数组

什么是数组？

数组其实就是一个容器 负责装元素 一组类型相同 存储空间连续的数据 数组的基本应用

创建数组对象：

int [] data = new int [5];//5: 空间大小

data[0] = 45;

data[1] = 33;…

int[] data = new int;

int[] data = {11,22,33,55,77};

如何得到数组的大小：

数组对象.length

System.out.println(data.length);

如何得到数组里面的某个元素：

数组对象 [下标] System.out.println (data [2]);

如何遍历数组：

for + 下标 这里的 x 永远是 int 类型 因为表示下标

foreach 这里的 x 类型看数组的类型

​ for (数组类型 x : data)

​ for (char x : zi){ // 其中 x 的单位取决于 zi 的单位

# 数组的复制

说明：在 java 中，数组是引用类型变量，可以看成是一个对象，在创建一个数组时，和创建一个新的对象时同样的道理 (毕竟创建时使用 new 关键字了)，这个对象存放在堆内存中，栈内存中只是存放了它的位置索引 (即在堆内存中的位置，相当于栈中放了它的门牌号)，这样在 test 方法中引用了数组变量，同样是通过栈中存放的位置索引找到这个变量在堆内存中的位置，然后进行修改操作，test 方法可不是将它重新压到栈中再修改的，是直接在堆中改，直接在堆中改，直接在堆中改 (重要的事情说三遍)，然后很淡定的退出了 test 方法，它是退出没事了，堆中的数组变量已经被不可逆转的修改了，因此 test 方法前和方法后打印出来的数组内容是不一样的。

三种有各种问题的复制方式:

错误的复制方式 只是内存里面的对象多了一个引用而已

笨重的复制方式 需要程序员挨个的进行手动复制

调用 "克隆" 方法 不符合正常的应用开发 [数组.clone，如 int [] temp = data.clone ();]

​ int [] temp = data.clone (); // 这时候 temp 和 data 不是一个地址

System.arraycopy(1,2,3,4,5);

- 1: 要复制的老数组对象

- 2: 老数组的下标几开始复制

- 3: 要复制到的新数组对象

- 4: 新数组的下标几开始存储

- 5: 要复制的长度

- 如果新老数组是同一个数组对象的话 可以实现的是一个数组的前后移动

- 优势：从任意下标开始复制

Arrays. copyOf (老数组对象，新数组长度);

- 优势：有返回值 可以一边复制 一边接受新数组对象

- 劣势：永远只能从下标 0 开始复制 -》 扩容 缩容

- 需要导包：import java.util.*;

# 面向对象的三大特点：封装 继承 多态

封装

什么是封装？

封装用 private 将不想被外界访问到的属性 / 方法隐藏起来

private: 私有的 只有本类可以访问

public: 公共的 谁都可以访问的

封装的好处？

类的开发者：数据得到了保护 从而更加的安全

类的使用者：将精力放在核心业务逻辑上

封装的要求？

对属性的要求：每个属性前面都需要加上 private 每个属性再配套提供两个 public 修饰的方法：

​ 一个是用来给属性赋值的方法 -》 setter public void setXXX (参数)

​ 一个是用来获得属性值的方法 -》 getter public 数据类型 getXXX ()

封装方法的要求：

判断哪些是核心方法，用 public 修饰

哪些是服务于核心方法，用 private 修饰

若核心方法要求太大，可以分出去

私有的自己访问可以，别人访问不行

继承

用 extends [派生] 实现两个类之间 is a 的关系 [继承关系]

继承的作用？

共享代码

继承的缺点？

java 中的类只能单根继承

无法继承封装的属性，但可以使用 public 方法 [setter 修改]

–java 类只能单根继承 [即只能继承一个类]

多态

一个对象总有不同的类型去定义它

作用一：创建对象

父类类型 = new 子类对象 ();

Animal xx = new Cat();

Person x = new Teacher();

当我们拿着多态创建完一个对象之后 这个对象只能调用父类和子类共有的属性 / 方法 一旦调用子类 特有的属性 / 方法 -> 报错

–编译器看创建等号前面，运行时才看等号后面 new 后面【对象】

–编译和运行都看等号左边【属性】

作用二：放在参数里面用于解除代码之间的耦合度 [放参数里也就相当于多态：形参 对象名 = new 实参 ()]

如：

高内聚 低耦合

a.getClass ().getName (); 通过对象得到其类名 [new 的类名]

# 方法覆盖 override

方法覆盖的条件：

需要发生在有继承关系的两个类中的子类中 [即子类在继承父辈的方法时，想修改，便可以在子类里重新实现一下]

public | void test(int x) | throws Exception{}

脑袋 | 躯干 | 尾巴

脑袋：子类的访问权限修饰符 >= 父类的权限

躯干：返回类型 + 方法签名 一模一样 [jdk5.0 之前][jdk5.0 之后，协变返回类型，即返回类型可以换成原返回类型的继承子类]

尾巴：异常处理 <= 父类异常 [范围]

​ 异常：程序执行的过程中 可能会出现的隐患

权限：

public

protected

default [friendly][默认]

private [无法继承，所以无法覆盖]

Exception: 异常

处理方式：throws: 抛出

try catch: 自行处理

@Override// 注解 表示下面的方法一定要覆盖父类的某个方法 [让电脑检擦一遍]

# 构造方法

构造方法是创建对象的时候调用的方法 -> 收尾的工作

构造方法的作用？

构造方法语法的出现是为了方便程序员开发 可以在创建对象的同时直接给属性赋值

在 java 中只要是个类就一定有构造方法 即使我们没有写 系统也会提供一个默认的构造方法 [默认的构造方法：无参 空体]

如果我们想要在创建对象的同时直接给属性赋值的话 需要自己写一个构造方法

一旦我们写出自己的构造方法之后 默认的构造方法将不再提供

构造方法的特点

- 没有返回类型 连 void 都不能写 -> 修饰符 + 方法签名 (){}

- 构造方法的名字需要和类名一模一样 -> 构造方法首字母一般大写

构造方法的首行

super ()[默认]: 表示要执行本构造方法之前 先去执行父类的构造方法

​ 默认找父类的无参构造方法

​ 假如父类没有无参构造方法 提供父类的无参构造方法

​ 在 super () 里面传参数 指定他找父类的哪一个构造方法

this (): 要执行本构造方法之前 先去执行本类的其他的构造方法

​ 具体执行本类的哪一个构造方法 看参数类型

- 构造方法不能被子类继承得到

但构造方法首行都默认有 super ();[object 没有]

底层会先造父类对象

当父类的构造有参数时，子类的构造方法必须要对应写 super ()[除非有 this (), 但 this () 到最后还是有 super ()]，因为这时候原本的默认构造方法就没用了 [也可以自己在父类再重载一个无参无体的构造方法]

this () 调用本类其他构造方法

this 和 super 一个构造方法只能有一个，因为都得在首行

a.equals (“b”); 用于 String 类型判断 a 的内容是否与 b [字符串] 相同

# 面试

Override 和 Overload 之间的区别？

含义不同

​ Override: 方法覆盖

​ Overload: 方法重载

发生的位置不同

​ Override: 需要发生在有继承关系的两个类中

​ Overload: 需要在同一个类体中

对返回类型的要求不同

​ Override: 分版本

​ jdk5.0 之前 一模一样 jdk5.0 开始 协变返回类型

​ Overload: 对返回类型没有要求

对参数的要求不同

​ Override: 参数必须相同

​ Overload: 参数必须不同 [类型 个数 顺序]

作用不同

Override: 让一个方法变得更加优秀

Overload: 同时满足用户的不同需求

方法重载需要哪些条件？

需要发生在同一个类体中

方法名需要一模一样

方法的参数需要不同

类型不同 个数不同 顺序不同 方法重载对修饰符 和 返回类型没有要求

方法覆盖的条件？

需要发生在有继承关系的两个类中 而且是在子类里面发生的 [子类在继承得到父类的某些方法之 后 觉得父类的实现不好 于是在子类里面重新实现一下]

构造方法能不能方法重载？

可以

构造方法能不能方法覆盖？

不能方法覆盖 方法覆盖的前提是需要前继承得到

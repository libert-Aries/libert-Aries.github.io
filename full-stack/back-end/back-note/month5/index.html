# @RequestMapping 注解

  • 可以用在接口、类、方法
    1. name：映射名称
    2. String [] path：请求的 URL 的地址
    3. String [] value：请求的 URL 的地址
    4. ReqeustMethod [] method ：设置请求方法
    5. String [] produces：设置响应头的 Content-Type（设置响应结果的类型）
    6. String [] consumes：设置请求头的 Content-Type（只接收哪种参数类型）
    7. String [] params：设置请求参数，请求参数中必须携带哪个参数、必须没有什么参数、参数值是什么
    8. String [] headers：设置请求头，请求头中必须携带哪个请求头、必须没有什么请求头、请求头值是什么

# Spring 容器和 Spring MVC 容器之间的关系

  1. Spring 容器通常称为父容器，最初用来整合 Struts 等 Web 框架；

  2. Spring MVC 容器通常称为子容器；

  3. Spring MVC 容器可以使用 Spring 父容器的 Bean（Service 可以注入给 Controller）

  4. Spring 父容器不能使用 Spring MVC 容器的 Bean（Controller 不能注入给 Service）

  5. 这两个容器不能使用同一个配置文件，并且需要扫描不同的包

  6. 如何配置 Spring 父容器

    ContextLoaderListener 加载名称为 contextConfigLocation 的 context-param 参数，

    如果不配置这个参数，默认使用 /WEB-INF/applicationContext.xml 作为 Spring 父容器

# Spring 的声明式事务配置 - 注解配置

  1. Spring 注解事务配置

  2. @Transactional 注解属性说明

  3. isolation => Isolation (5 种) - 隔离级别

  4. timeout => int: 秒为单位，-1 表示不超时；5: 表示 5 秒超时

  5. readOnly => boolean 用于加速查询效率

  6. rollbackFor => Class [] 表示哪些异常需要回滚

  7. rollbackForClassName => String [] 填写要回滚的异常的全限定类名称

  8. noRollbackFor => Class [] 表示哪些异常不需要回滚

  9. noRollbackForClassName => String [] 填写不需要回滚的异常的全限定类名称

  10. transactionManager => String 指定事务管理器 bean 的 id

  11. propagation => Propagation 事务传播行为

  12. Spring 事务的隔离级别

TransactionDefinition 接口中定义了五个表示隔离级别的常量：

  • DEFAULT ：默认使用数据库的隔离级别
  • READ_UNCOMMITTED
  • READ_COMMITTED
  • REPEATABLE_READ
  • SERIALIZABLE

注意：这里主要说一下 DEFAULT 配置，这是默认值，表示使用底层数据库的默认隔离级别。对大部分数据库而言， 通常这值就是 ISOLATION_READ_COMMITTED。

  1. Spring 的支持七种传播特性

  2. REQUIRED：表示如果当前存在一个事务，则加入该事务，否则将新建一个事务；这个是 Spring 事务默认传播 行为

  3. REQUIRES_NEW：表示不管是否存在事务，都创建一个新的事务，把原来的事务挂起，新的事务执行完毕， 继续执行老的事务；

  4. SUPPORTS：表示如果当前存在事务，就加入该事务；如果当前没有事务，那就不使用事务；

  5. NOT_SUPPORTED：表示不使用事务；如果当前存在事务，就把当前事务暂停，以非事务方式执行；

  6. MANDATORY：表示必须在一个已有的事务中执行，如果当前没有事务，则抛出异常；

  7. NEVER：表示以非事务方式执行，如果当前存在事务，则抛出异常；

  8. NESTED：这个是嵌套事务； 如果当前存在事务，则在嵌套事务内执行； 如果当前不存在事务，则创建一个新的事务； 嵌套事务使用数据库中的保存点来实现，即嵌套事务回滚不影响外部事务，但外部事务回滚将导致嵌套事务 回滚；

  9. Spring 事务的回滚规则

  10. Spring 事务默认只对 运行时异常 [非检查型] 起作用（也就是出现运行时异常时，回滚数据）；

  11. Spring 事务默认对 非运行时异常 [检查型异常、编译期异常] 不起作用；

  12. 可以使用 rollbackFor 指定 Spring 事务对哪些异常起作用；

  13. 可以使用 noRollbackFor 指定 Spring 事务对哪些异常不起作用；

CGLIB 是一个强大的、高性能的代码生成库，它可以在运行期继承 Java 类或实现 Java 接口，被广泛应用于 AOP 框 架中，用以提供方法拦截操作。

# Spring AOP - Aspect Oriented Programming - 面向切面编程

# Spring AOP 的概念

  1. Join point 连接点：一个方法执行（方法）
  2. Pointcut 切入点：连接点的集合，用于圈定连接点的范围
  3. Advice 通知、增强：在连接点上执行一个动作（代理对象执行逻辑）
  4. Aspect 切面：连接点和通知组成的（在切入点约束下连接点上执行通知）

# 通知的类型

  1. Before advice：前置通知，连接点执行前执行
  2. After returning advice：连接点执行成功之后执行的通知
  3. After throwing advice：连接点抛出异常之后执行的通知
  4. After (finally) advice：不管连接点是否执行成功，都会执行的通知
  5. Around advice：环绕通知，可以控制连接点的执行流程

# Spring SPI 机制

# Spring SPI

Spring 框架从 3.2 版本开始提供了 SpringFactoriesLoader 工厂的加载机制，类似于 Java 提供的 SPI 机制，Spring 利用这种机制实现了 Spring Bean 的自动配置。

# SpringFactoriesLoader 工厂的加载机制的默认约定

  1. 在 classpath 根目录下创建 META-INF 目录

  2. 在 classpath:/META-INF/ 目录下创建 spring.factories 文件

    文件内容：

    key1=v1,v2

    key2=v3,v4

# Spring Boot 的自动配置

  1. 启动类上的注解 @SpringBootApplication 由 @SpringBootConfiguration、@EnableAutoConfiguration 和 @ComponentScan 组成的，其中最重要的注解是 @EnableAutoConfiguration

  2. @EnableAutoConfiguration 注解导入一个 AutoConfigurationImportSelector 类型，这个类实现了 ImportSelector 接口，并实现了这个接口中的 selectImports()方法 ，在方法体中利用 Spring 的工厂加载机制加载 classpath:/META-INF/ 目录下 spring.factories 文件中以 EnableAutoConfiguration 为 key 所有 value 值表示的类型，对这些类型进行 有条件的 自动配置；

# Spring Boot 2.7.0 之后的自动配置

  1. 自动不再读取 classpath:/META-INF/ 目录下 spring.factories 文件

  2. 现在是读取 classpath:/META-INF/spring 目录下的 org.springframework.boot.autoconfigure.AutoConfiguration.imports 文件

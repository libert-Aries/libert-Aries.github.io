# 面试题（&& 和 & 之间的区别？）

相同点：&& 和 & 都叫做逻辑的与 相当于汉文中并且的意思

需要前后两个条件都成立才能执行

不同点：&& 叫做逻辑的短路与 有短路特性

短路特性：如果通过前面的条件可以得出最终结果的话 后面的条件直接舍弃 不在判断

|| 和 | 之间的区别？

相同点：|| 和 | 都是逻辑的或 相当于汉文中或者的意思

需要前后一个条件成立就能执行

不同点：|| 叫做逻辑的短路或 有短路特性

如果通过前面的条件可以得出最终结果的话 后面的条件直接舍弃 不在判断

  • 重点

# 面试题（成员变量和局部变量之间的区别？）

  • 定义的位置不同

    • 成员变量：定义在类体里面
    • 局部变量：定义在方法体里面

  • 作用范围不同

    • 成员变量：也叫属性 表示对象有什么 依赖于对象存在当我们创建对象 [new] 的时候 属性伴随着也会创建 当消亡对象的时候 属性伴随着也会消亡
    • 局部变量：从定义的那一行开始 一直到所在的方法体执行结束之前可以访问 一旦所在的方法体执行结束 局部变量立即消亡 依赖于方法存在

  • 默认值不同

    • 成员变量：即使不赋值也有默认值
    • 局部变量：没有默认值 要求在使用之前必须先赋值

    成员变量和局部变量可以重名

    当成员变量和局部变量重名的时候 直接访问这个变量默认是局部

    如果想要访问成员变量 需要在变量前面加上 this.

# 面试题（Override 和 Overload 之间的区别？）

  • 含义不同
    • Override: 方法覆盖
    • Overload: 方法重载
  • 发生的位置不同
    • Override: 需要发生在有继承关系的两个类中
    • Overload: 需要在同一个类体中
  • 对返回类型的要求不同
    • Override: 分版本
      • jdk5.0 之前 一模一样
      • jdk5.0 开始 协变返回类型
    • Overload: 对返回类型没有要求
  • 对参数的要求不同
    • Override: 参数必须相同
    • Overload: 参数必须不同 [类型 个数 顺序]
  • 作用不同
    • Override: 让一个方法变得更加优秀
    • Overload: 同时满足用户的不同需求

# super () 和 this () 的区别

默认：super (): 表示要执行本构造方法之前 先去执行父类的构造方法

  • 默认找父类的无参构造方法
  • 假如父类没有无参构造方法
    • 提供父类的无参构造方法
    • 在 super () 里面传参数 指定他找父类的哪一个构造方法

this (): 要执行本构造方法之前 先去执行本类的其他的构造方法

  • 具体执行本类的哪一个构造方法 看参数类型

# static 修饰符

  • static 可以修饰什么

    • 属性：静态属性 表示整个类型共享一份的属性 不是每个对象都有一份的属性 需要拿着类名去调用

    • 方法：静态方法 需要拿着类名对象去调用 静态方法里面只能直接的访问静态的成员 如果想要在静态方法里面访问非静态成员 需要创建对象 拿着对象调用

    普通方法需要拿着对象去调用

    • # 面试题：普通方法和静态方法谁调用简单？

      静态方法 直接拿着类名去调用

    • 既然静态方法调用简单 开发的时候为什么不把一个类所有的方法都写成静态的？

      静态方法里面只能直接的访问静态成员 如果想要在静态方法里面调用非静态的成员 需要创建对象 拿着对象去调用

  • 代码块：初始化一个普通属性的

    当我们想要初始化一个静态属性的话 需要使用静态代码块 -》 static {}

    普通代码块给普通属性赋值的 执行时间：当创建对象的时候执行 创建几个对象执行几次

    静态代码块给静态属性赋值的 执行时间：当类第一次被加载的时候执行 从头到尾只执行一次

# 面试题（static 为什么不能修饰局部变量？）

# 面试题（ArrayList 和 LinkedList 之间的区别？）

  • ArrayList 和 LinkedList 底层数据结构不同 导致他们的优劣势不同
  • ArrayList 底层基于数组实现的
    • 优点：遍历查找 随机访问效率高
    • 缺点：添加 / 删除元素效率低
  • LinkedList 底层基于链表实现的

    • 优点：添加、删除元素效率高

    • 缺点：遍历查找 随机访问效率低 永远从下标 0 开始找起

    • 在项目开发的时候 尽量避免使用 LinkedList 的 get (下标) 方法

# 面试题（HashMap 和 Hashtable 之间的区别？）

同步特性不同

​ Hashtable 同一时间允许一个线程进行访问 效率较低 但是不会出现并发错误

​ HashMap 同一时间允许多个线程进行访问 效率较高 但是可能会出现并发错误

​ jdk5.0 开始 集合的工具类里面提供一个方法 将线程不安全的 HashMap 变成线程安全 Map 集合 所以 Hashtable 渐渐被淘汰了

​ Map map = Collections.synchronizedMap (HashMap 对象);

​ 对 null 的处理不同

​ Hashtable 无论是主键还是值对象 都不能传 null 否则触发空指针异常

​ HashMap 无论是主键还是值对象都可以传 null 但是由于主键是唯一的 所以主键只能

​ 传一个 null 但是值对象可以传多个 null

​ 分组组数不同

​ Hashtable 默认分 11 个小组 程序员可以随意的指定

​ HashMap 默认分 16 个小组 程序员可以随意的指定 但是最终一定会变成 2 的 n 次方数

​ 出现的版本不同

​ Hashtable since jdk1.0

​ HashMap since jdk1.2

# 面试题（HashMap Hashtable ConcurrentHashMap 之间的区别？）

Hashtable 为了追求安全性 当某个线程进入哈希表某个小组处理元素的时候 会将整个哈

​ 希表加锁 所以同一个时间最多只能一个线程操作哈希表

​ HashMap 为了追求高效性 将不再加锁 可能会出现的问题是一个线程在处理某个小组的

​ 时候又来一个线程同时处理该小组 可能会引发并发问题

​ ConcurrentHashMap 安全性和高效性都介于 Hashtable 和 HashMap 之间

​ 采用了锁分离的机制 当某个线程处理某个小组的时候 只是将该小组加锁 如果又来一个

​ 线程想要处理该小组 需要等待 如果想要处理其他的小组 可以直接处理

# 面试题（HashMap 在 jdk7.0 之前和 jdk7.0 之后的变化）

jdk7.0 之前 哈希表底层基于数组 + 链表组成的

​ 数组：用来装表头信息 为了方便元素更加快速的找到应该去到哪一个小组

​ 组内采用是链表数据结构进行添加 / 删除元素

​ 7.0 之前 采用头插法

​ jdk7.0 开始 哈希表底层数组 + 链表 + 二叉树实现的

组内采用的是链表和二叉树实现

​ 如果组内元素 <= 7 采用链表实现

如果组内元素 > 7 个元素 将链表改成二叉树 [为了效率高]

jdk7.0 开始 采用尾插法

  • ArrayList5: 要被删除的对象会主动调用他自己的 equals 方法和集合里面的元素作比较

  • ArrayList6: 当我们使用迭代器遍历集合的时候 不允许对集合的整体进行添加 / 删除

    否则触发 CME 异常 如果在遍历的时候 想要删除的话：car.remove ()

  • ArrayList7:ArrayList 类里有两个常见的构造方法

    ArrayList list = new ArrayList (int 数组容量);

    ​ 容量 < 0 抛出异常

    ​ 容量 >= 0 写几开辟几块空间

    ArrayList list = new ArrayList ();// 默认开辟 10 块空间

    集合无论底层开辟多大空间 都可以装无数个元素 集合会自动扩容

    list.ensureCapacity (int 容量)：扩容

    list.trimToSize (): 缩容

# 面试题（ArrayList 和 Vector 之间的区别？）

  • 同步特性不同
    • Vector 同一时间允许一个线程进行访问 效率较低 但是不会出现并发错误
    • ArrayList 同一时间允许多个线程进行访问 效率较高 但是可能会出现并发错误
    • 从 jdk5.0 开始 集合的工具类里面提供一个方法 将线程不安全的 ArrayList 变成线程安全的 List 集合 于是 Vector 渐渐被淘汰了
    • List 对象 = Collections.synchronizedList (ArrayList 对象)
  • 扩容机制不同
    • Vector 分构造方法
      • Vector (int 大小)-》2 倍扩容 Vector (8) -> 8 16 32…
      • Vector (int 大小，int 步长)-》定长扩容 -》 Vector (10,3) -> 10 13 16…
    • ArrayList 分版本
      • jdk6.0 及之前 x * 3 / 2 + 1
      • jdk7.0 及之后 x + (x>> 1)
    • 出现的版本不同
      • Vector since jdk1.0
      • ArrayList since jdk1.2

# 面试题（ArrayList 和 LinkedList 之间的区别？）

  • ArrayList 和 LinkedList 底层数据结构不同 导致他们的优劣势不同
  • ArrayList 底层基于数组实现的
    • 优点：遍历查找 随机访问效率高
    • 缺点：添加 / 删除元素效率低
  • LinkedList 底层基于链表实现的
    • 优点：添加、删除元素效率高
    • 缺点：遍历查找 随机访问效率低 永远从下标 0 开始找起
    • 在项目开发的时候 尽量避免使用 LinkedList 的 get (下标) 方法

计数器 modCount

modCount: 记录对集合的处理 [添加 / 删除…] 次数

# 面试题（ArrayList LinkedList Vector Stack HashSet TreeSet 底层数据结构）

数组 双向循环链表 数组 栈 哈希表 二叉树

# 面试题（ArrayList 和 HashSet 在存取特性上有什么不同）

ArrayList: 有序 不唯一 遍历 for + 下标 foreach 迭代器

HashSet: 无序 唯一 遍历 foreach 迭代器

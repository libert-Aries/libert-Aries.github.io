# 第一周总结

Java 跨平台功臣：JVM–java 虚拟机

JVM = 秘书 + 保镖 + 翻译 = 类加载器 + 字节码校验器 + 解释执行器

  • 秘书：类加载器 ClassLoader----- 加载一个.class 文件

  • 保镖：字节码效验器 ByteCode

    Verifier----- 挨个的解释每个字节码是否有问题。比如单词错误
    语法错误 …

  • 翻译：解释执行器 Interpreter----- 把.java [源文件] 翻译为.class [字节码文件]

-Java 应用系统跑在 JVM 虚拟机上而不是操作系统上

而像 C 之类的应用程序是直接跑在操作系统上的（详见图），这也就造成 C 写的程序移植到其他操作系统可能不能运行

ctrl+1：生成.class 文件

ctrl+2：类加载器运行.class 文件并弹出运行窗口

全局变量和局部变量的区别：

  • 定义位置不同：

成员变量：类中方法外

分为类变量（static 修饰）、实例变量（没有使用 static 修饰）

局部变量：方法内、代码块内

  • 默认值不同：

成员变量：有默认初始化值

局部变量：没有默认初始化值，必须显式赋值，方可使用

  • 作用范围不同：

成员变量：

静态变量（类变量），在本类中随便用，在其他类中使用 "类名。静态变量"

非静态变量（实例变量），在本类中只能在非静态成员中使用，在其他类中使用 "对象名。非静态的变量"

局部变量：方法内、语句块内

  • 可用修饰符不同：

成员变量：权限修饰符、static、final 等

局部变量：不能用权限修饰符修饰，可以用 final 修饰

  • 内存中位置不同：

成员变量：

静态变量（类变量）：方法区

非静态变量（实例变量）：堆

局部变量：栈

  • 生命周期不同：

成员变量：

静态变量（类变量）：和类相同

非静态变量（实例变量）：和所属的对象相同，每一个对象是独立的

局部变量：从变量初始化到变量所在的方法或语句块执行结束

变量使用注意：

局部变量定义后，必须显式赋值才能使用，因为系统不会为局部变量执行初始化操作。这就意味着，定义局部变量后，系统并没有为这个变量分配内存空间。直到程序为这个变量赋值时，系统才会为局部变量分配内存，并将初始值保存到该内存中

基本数据类型局部变量：直接把这个变量的值保存到该变量所对应的内存中。

引用数据类型局部变量：变量对应内存中保存的是地址，通过该地址引用堆中的数据

栈内存中的变量无需垃圾回收器回收，其往往随方法或代码块的运行结束而结束

jdk 和 jre 之间的区别？

jdk 表示 java 软件开发工具包

如果一台电脑上面安装 jdk，既可以编写代码，又可以运行代码

jre 表示 java 运行环境

如果一台电脑上面只安装 jre，只能运行代码

以下哪些是合法的标识符？A B

A:String [类名] B:Int

C:const [保留字] D:Hello World

java 中的数据类型分为几种？

2 种 / 无数种

基本数据类型 [简单数据类型][程序里面自带的数据类型]

和引用数据类型 [自定义数据类型][比如 student 类]

基本数据类型分为几种？

布尔类型 Blooean

字符类型 char [String 不是基本数据类型]

整数类型 byte short int long

浮点类型 double

String 类属于基本数据类型还是引用数据类型？

引用数据类型

String 类 Sun 个数开发好的类 自动会导入每篇代码里面 [在 lang 包里]

下列输出？

short x = 1; //-32768 32767

x = x + 1; //short = int

System.out.println (x); // 报错 可能损失精度

short y = 1;

y += 1;y = (short)(y + 1);

System.out.println(y);//2

2.0 - 1.1 结果是多少？

反正不是 0.9

为什么会得到一个错误的结果？

java 中任何一个整数类型都可以用二进制

精确的表达出来 但是不代表任何一个小数

也可以在有限的位数里面 表达出来

1.1 在计算机底层存储的时候 出现了数值位的截取

导致数字不正确 所以结果也不正确

怎么解决？

公交卡 单位 -》 1000 - 7 - 7

数字扩大 100 倍 单位缩小 100 倍

BigDecimal: 大数值 解决小数精度问题

&& 和 & 之间的区别？

相同点：&& 和 & 都叫做逻辑的与 相当于汉文中并且的意思
需要前后两个条件都成立才能执行

不同点：&& 叫做逻辑的短路与 有短路特性
短路特性：如果通过前面的条件可以得出最终结果的话 后面的条件直接舍弃
不再判断

例子：如果晚上我有空的话 而且你们愿意 我请大家吃饭 if (我有空 &
你们愿意)

例子：两个人去领结婚证 -》两本户口本 && 人 && 9.9 元钱

|| 和 | 之间的区别？

相同点：|| 和 | 都是逻辑的或 相当于汉文中或者的意思
需要前后一个条件成立就能执行

不同点：|| 叫做逻辑的短路或 有短路特性

如果通过前面的条件可以得出最终结果的话 后面的条件直接舍弃 不在判断

例子：火车站买票 -》 出示证件

身份证 || 户口本 || 驾照 || 护照…

Java 没指针

Java 回收内存自动回收

程序执行后，底层只找 main 方法，是程序的入口

面向对象需要有对象

对象要用类创建

java 分大小写和全半角

(char)(Math.random()*26+65)

所有 class 修饰的类都是引用数据类型

"+" 只有遇到 String 才是追加连接，其他 [包括 char] 都是加法

1 + ‘1’ + “1” 结果是 501

int char String

前两个在计算时，后面的 char 强制转换为 int 型 [即 1 对应的 ASCII 码]，值为 49

在和最后一个 String 类型相 + 时，为追加连接

类似 System.out.println (45> ‘a’);

45 和’a’做比较时，char 类型转换为 ascII 码的 int 类型，再做比较

不同类访问属性、方法时需要用对象访问

char 类型无法强制转换为 String 类型，但可以在 char 前面先写一个 ""

如：a [String]=""+b [char]; 就可以将 b 转换成 String 再赋值给 a

整数除以整数结果还是整数，即整数 / 整数 = 整数

注意运算符：100/8*8 100 以内 8 的最大倍数

还可以 100-100%8

一个属性依赖于哪个对象，取决于谁调用 show ()

在同一个类里调用属性 x，相当于 this.x, 其中 this. 表示当前调用该方法的对象

所有运算符的最小处理单位是 int 类型，即所有运算结果都默认 int 类型

byte a=1,b=2;

byte c=a+b; 报错

小数不能使用 ^[异或]，因为小数不安全

语句高于运算符

所以不能在运算符里用语句

比如不能在三目运算符里写打印

负数的二进制表示是补码形式

如二进制 10011001 表示的不是 - 25，而是 - 103

将二进制补码转换为原码时，可以将其和 - 1 [11111111] 的补码进行比较

如果有和 - 1 补码不一样的位 [即 0 的位]，就用 - 1 减去对应位的正数值

如 11100110

11111111==>-1-1-8-16=-26

方法不能嵌套

当我们写的方法和 main 方法在同一个类体的时候

这个方法加 static

当我们写的方法和 main 不在一个类体的时候

不加 static

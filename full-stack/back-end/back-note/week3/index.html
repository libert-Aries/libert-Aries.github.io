# String 的 20 个方法

–String 类里面的所有方法都不会直接处理原本的字符串，而是将符合条件的字符串返回，所以注意接收

方法 返回值 作用

length () int 得到 String 里的字符个数

和切开有关的方法 -----------------------------------------------------------

getBytes () byte [] 将一个字符串转换成字节数组【ASCII 码】

​ 【一个英文对应一个字节，一个中文对应两个字节】

​ 【中文对应的字节二进制都是 1 开头的】

toCharArray () char [] 将字符串打散，转换成字符数组

split (String) String [ ] 将字符串按括号 (String) 劈开

​ 【遇见就劈，没遇见就将原本的字符串以单数组返回】

​ 【若劈开元素在开头，则劈开后数组开头有一个空数组元素，结尾没有】

和判断有关的方法 ----------------------------------------------------------

equals (String) boolean 判断两个字符串的内容是否一模一样

​ 【区分大小写，比较严格】

​ 【可以固定内容在前面，这样可以有效的避免空指针异常】

equalsIgnoreCase (String) boolean 忽略大小写的比较两个字符串的内容是否一样

contains (String) boolean 判断字符串里面是否有指定内容

startsWith (String) boolean 判断字符串是否以指定内容开始

endsWith (String) boolean 判断字符串是否以指定内容结尾

和改变内容有关的方法 -------------------------------------------------------

replace (String,String) String 将一个字符串里面指定内容全部替换成另一个指定内容

replaceAll (String,String) String 将一个字符串里面指定内容全部替换成另一个指定内容

​ 【支持正则表达式】

replaceFirst (String,String) String 将一个字符串里面第一次出现的内容替换成指定的内容

trim () String 将字符串的前后空格删除

toUpperCase () String 将字符串全部转换成大写

toLowerCase () String 将字符串全部转换成小写

substring (int x,int y) String 从下标 x 截取到下标 y 的前一个元素

​ 【注意大小写特殊】

substring (int x) String 从下标 x 截取到最后

和位置有关的方法 -----------------------------------------------------------

charAt (int) char 找到某个下标对应的元素

indexOf (String) int 找到某个内容第一次出现的下标

lastIndexOf (String) int 找到某个内容最后一次出现的下标

# 四种权限修饰符

public: 公共的 表示谁都可以使用的

protected: 受保护的 只有本包 [文件夹] 内可以访问 + 包外有继承关系的子类【直系子类】中可以访问

default: 默认的 只有本包内可以访问

private: 私有的 只有本类可以访问

各自可以修饰什么

public 类 成员 [属性 / 方法]

protected 成员 [属性 / 方法]

default 类 成员 [属性 / 方法]

private 成员 [属性 / 方法]

---------------------------------------------------------------------------

# static 修饰符

属性：静态属性，表示整个类型共享一份的属性 不是每个对象都有一份的属性 需要拿着类名去调用

​ 比如：System.out [static] System.in[static]

方法：静态方法 需要拿着类名对象去调用 静态方法里面只能直接的访问静态的成员 如果想要

在静态方法里面访问非静态成员 需要创建对象 拿着对象调用。依赖于类

普通方法需要拿着对象去调用

代码块：给普通属性赋值，创建对象的时候执行，初始化一个普通属性的

​

当我们想要初始化一个静态属性的话 需要使用静态代码块 -》 static {}

普通代码块给普通属性赋值的 执行时间：当创建对象的时候执行 创建几个对象执行几次

静态代码块给静态属性赋值的 执行时间：当类第一次被加载的时候执行 从头到尾只执行一次

【创建一个对象的时候会自动调用类里的普通代码块，而用类名的时候会自动调用类里的静态代码块且只调用一次】

【代码块相当于方法体】

---------------------------------------------------------------------------

# final 最终的 修饰符

final 能修饰哪些东西？

类：最终类，不能有子类，但可以有父类

​ 【String 和 Math 类都是 final 修饰的】

方法：最终方法，只能被继承，不能被覆盖

变量 [属性 + 局部变量]：最终变量，常量，一旦赋值就不能修改

​ 【修饰引用数据类型保护的是地址而不是数值】

---------------------------------------------------------------------------

# abstract 抽象类 修饰符

类：抽象类 表示这个类型不形象不具体 不能创建对象

方法：抽象方法 表示这个类型一定会这个方法，但现在不给出具体的实现，具体实现看子类

–Java 中的类只要存在抽象方法 那么这个类就一定要变成抽象类

–抽象类里面既可以定义抽象方法 也可以定义普通方法

# Object 类

toString()

制定用引用数据类型打印出来的东西

如果直接打印类名会打印 “类 @地址映射”

equals()

制定一个类型的比较规则

当一个类继承 Object 类获得 equals () 方法之后 如果不覆盖的话 默认和 Object 类型保持一致 Object 类只要地址一样就视为相等对象

【object 里比的是地址 (用的 ==)】

hashCode()

生成一个对象的散列特征码 [哈希码值]

散列：将一大组数据分散为不同的小组

当方法 [equals] 被重写时，通常有必要重写 hashCode 方法，以维护 hashCode 方法的常规协定，该协定声明相等对象必须具有相等的哈希码。

即 x.equals (y) -> true x.hashCode () 等于 y.hashCode ()

当一个类继承得到 Object 类里面的 hashCode () 方法 在没有覆盖的时候 默认通过地址生成哈希码值 如果不想通过地址生成 可以覆盖 hashCode ()

【String 的为 str [0]*31(str.length-1)+str[0]*31 (str.length-1)…】[31 是个很神奇的数字]

clone()

"克隆" 对象的方法 -> protected: 受保护的

【克隆的类需要接口 Cloneable】

finalize()

对象的 "遗言" 方法 当一个对象要被 gc 回收的时候 会主动执行的一个方法

引用类型也可以强转【要在子父类之间转】

# ArrayList【底层是 Object 数组】

・特点：有序、不唯一

・数据结构：数组

包装类：boolean char byte short int long float double

​ Boolean Character Byte Short Integer Long Float Double

一、转包装类：将基本数据类型转换为引用数据类型【Integer 的 equals 比较的是内容】

int x = 45; Integer y = x; //5.0 开始 自动打包 封箱

Integer x = new Integer (45); int y = x; // 自动解包 拆箱 可以直接打包 解包

–Integer 类会自动缓存 - 128 到 127 之间的数字

二、包装类里面提供一个方法 可以将 字符串 转换成对应的基本数据类型

String str = “1234”; int x = Integer.parseInt (str); // 转成谁就用谁的包装类 配合 String 的方法

String str = “45.5”;double x = Double.parseDouble(str);

转成其他的基本数据类型也都有 [除了 char]

基本用法和特点：

特点 [有序：添加顺序 不唯一：同一个元素可以存放多次]

创建对象：ArrayList <泛型> list = new ArrayList<>();

添加元素：

​ 一次添加一个元素：list.add (元素)

​ 一次添加多个元素：Collections.addAll (集合，元素，元素…);

如何得到集合大小：list.size ()

如何得到集合某个元素：list.get (下标)

如何判断集合里面是否存在某个元素：list.contains (元素)

如何遍历集合：

​ for + 下标

​ foreach

​ 迭代器

删元素：

list.remove (int 下标)

list.remove (Object 元素)

list.clear (): 清空集合对象

一个 remove () 方法只能删除一个对象（第一个）

ArrayList 类里面指定元素删除的方法 底层尊重 equals () 比较机制

ArrayList 类里面 contains (元素) 底层尊重 equals () 比较机制

即要删除 [搜索] 对象会调用自己的 equals () 方法和集合里的元素做比较，true 就删，false 就不删

当我们使用迭代器遍历集合的时候 不允许对集合的整体进行添加 / 删除

否则触发 CME 异常 如果在遍历的时候 想要删除的话：car.remove ()

CME 并发修改异常（ConcurrentModificationException）

# ArrayList 类里有两个常见的构造方法

​ 容量 >= 0 写几开辟几块空间

list.ensureCapacity (int 容量)：扩容

list.trimToSize (): 缩容

在项目开发的时候，尽量避免扩容，扩容一次要进行的步骤：

建立一个新的数组对象

将老数组里的元素复制到新的数组里

改变引用指向

回收老数组对象

继续添加元素

# 面试

》new 和不 new 之间的区别？/String 类创建对象的方式有几种？他们之间的区别？

String x = “OK”;

String y = new String(“OK”);

不 new 的方式涉及到常量池查找机制

》String 和 StringBuffer / StringBuilder 之间的区别？

String str = new String(“OK”);//[]:2

StringBuffer buffer = new StringBuffer(“OK”);//[]:2 + 16

StringBuffer 和 StringBuilder 在第一次创建对象的时候 底层会多预留 16 块缓冲区

缓冲区可以让追加连接效率高

String 类每次追加连接 底层都需要换一个新的数组对象

》StringBuffer 和 StringBuilder 之间的区别？

StringBuffer 同一时间允许一个线程进行访问 效率较低 但是不会出现并发错误 把子肉

StringBuilder 同一时间允许多个线程进行访问 效率较高 但是可能会出现并发错误 KFC

》普通方法和静态方法谁调用简单？

静态方法 直接拿着类名去调用

》既然静态方法调用简单 开发的时候为什么不把一个类所有的方法都写成静态的？

静态方法里面只能直接的访问静态成员 如果想要在静态方法里面调用非静态的成员 需要

创建对象 拿着对象去调用

比如：Math.random () Math.sqrt () Arrays.sort () System.arraycopy ()

Arrays.copyOf() System.currentTimeMillis()

》java 中变量大还是属性大？

变量【变量包括局部变量和属性】

》static 为什么不能修饰局部变量？

static 修饰的变量要求类一加载就要在内存里面找到他 但是局部变量只有当方法调用 代码执行到那一行的时候才能创建

一般类的加载在前面 方法调用在后面 这两个时间点赶不上一起 所以 static 不能修饰局部变量

》成员变量和静态变量之间的区别？

成员变量也叫属性 表示每个对象有什么 依赖于对象存在

静态变量也叫静态属性 表示整个类型共享一份的属性 依赖于类存在

》String 和 Math 类都是用 final 修饰的 为什么？

String 类叫做字符串类 是 java 中比较基础 比较核心的类

越是基础 越是核心的类 所有的程序员越应该保持一致

Math 类叫做数学类 里面定义的都是数学里面的公理 定理 不能随意的修改

》抽象类是类吗？

是

》抽象类有构造方法吗？

有 java 中只要是个类就一定有构造方法

> 抽象类是类 有构造方法 但是不能创建对象 抽象类里面的构造方法作用？

给子类构造方法首行的 super () 调用的

》final 和 abstract 能不能同时修饰一个方法？

不能 final 修饰的方法叫最终方法 不能被覆盖

abstract 修饰的方法叫抽象方法 需要子类去覆盖

矛盾

》抽象类和接口之间的区别？

java 中的两大类型：

抽象类：class

接口：interface

实现不同

抽象类：extends 单根继承

接口：implements 多重实现

里面定义的属性不同

抽象类：默认都是普通属性

接口：默认都是静态的最终变量 -》 public static final

里面定义的方法不同：

抽象类：既可以定义抽象方法也可以定义普通方法

接口：只能写抽象方法 但是从 jdk8.0 开始 接口里面允许出现普通方法了 但是必须加上 static

》== 和 equals 之间的区别？

== 是一个运算符 判断左右两边的值是否相等

基本数据类型 比较数值

引用数据类型 比较地址

equals () 判断两个对象能不能视为相等对象 程序员可以按照自己的意愿将两个对象视为相等对象 比

如：String 类 Sun 公司通过覆盖 equals () 只要两个字符串的内容一样视为相等对象

》Collection 和 Collections 之间的区别？

Collection 是所有单值集合统一的父接口 ： interface

Collections 是集合的工具类：class

》ArrayList 和 Vector 之间的区别？

同步特性不同

Vector 同一时间允许一个线程进行访问 效率较低 但是不会出现并发错误

ArrayList 同一时间允许多个线程进行访问 效率较高 但是可能会出现并发错误

从 jdk5.0 开始 集合的工具类里面提供一个方法 将线程不安全的 ArrayList 变成线程安全的

List 集合 于是 Vector 渐渐被淘汰了

List 对象 = Collections.synchronizedList (ArrayList 对象)

扩容机制不同

Vector 分构造方法

​ Vector (int 大小)-》2 倍扩容 Vector (8) -> 8 16 32…

​ Vector (int 大小，int 步长)-》定长扩容 -》 Vector (10,3) -> 10 13 16…

ArrayList 分版本

​ jdk6.0 及之前 x * 3 / 2 + 1

​ jdk7.0 及之后 x + (x>> 1)

出现的版本不同

Vector since jdk1.0

ArrayList since jdk1.2

》ArrayList 和 LinkedList 之间的区别？

ArrayList 和 LinkedList 底层数据结构不同 导致他们的优劣势不同

ArrayList 底层基于数组实现的

优点：遍历查找 随机访问效率高

缺点：添加 / 删除元素效率低

LinkedList 底层基于链表实现的

优点：添加、删除元素效率高

缺点：遍历查找 随机访问效率低 永远从下标 0 开始找起

在项目开发的时候 尽量避免使用 LinkedList 的 get (下标) 方法

–基本数据类型不会进堆内存

栈内存：引用 堆内存：对象

–foreach 相当于传参

–字节和字符的区别？

数组.length 是字节个数 英文一个占一个字节，中文一个占两个字节

【char 底层基于 Unicode 编码方式实现，不区分中英文，统一都是 2 个字节 16 位】

【windos 自动转换为 ASCII 码了】

String.length () 是字符个数 无论中英文都一个占一个字符

–static 不能修饰局部变量

–集合里自带一个属性：modCount：记录对集合的处理 [添加 / 删除…] 次数

迭代器会将 modCount 拷贝过来

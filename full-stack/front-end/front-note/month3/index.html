# html-css

  • form: 称之为表单，划定一个区域，在这个区域内的元素称之为这个表单的表单项，表单项可以收集

  • method: 表示提交表单时的方式，存在 get 与 post 两种提交方式

  • action: 表示用户在表单中的表单项里填写的内容最终提交到哪里，一般为后端地址

  • 单行文本输入框 type=“text” type=“password”

    • name: 表示提交键值对的键，可以随意书写

    • placeholder: 悬浮文本，当我们书写内容时消失，不书写时显示，用来进行简单的提示

    • required: 表示本表单项必填，如果不填则表单无法提交

    • autofocus: 表示自动获取焦点，存在浏览器差异性文本

    • value: 就是键值对的值，一般不书写，因为用户填写什么，value 就是什么

    • autocomplete=“off” : 关闭浏览器自动完成功能，否则会记住用户填写的值，之后显示在页面上

  • 单选框 type=“radio”

    • checked: 表示默认选中

  • 复选框 type=“checkbox”

  • 下拉列表框，下拉列表框的书写方式与其它表单项不同

  • 日期格式 type=“date”

  • 输入邮箱 注意这里仅能对邮箱格式进行简单的验证，如果想要填写后自动一封确认函等必须使用其它技术

  • 普通按钮 此按钮点击不会提交表单，必须搭配 js 绑定函数才可以激发

  • 提交和取消按钮

    • type=“submit” 点击此按钮后可以提交表单
    • type=“reset” 点击此按钮后可以重置表单
      • 由于是按钮，所以要书写 value 值，表示按钮上显示的文本，注意按钮本身无法传递参数

  • disabled: 每个元素都有这个属性，表示禁用本元素

  • a 链接

    • 就是一种简化版的表单，使用率较表单高得多，链接只有 get 一种提交方式
    • href: 就是链接提交的目的地，支持相对路径和绝对路径

    ​ 注意 链接自带样式，蓝色字体，自带下划线

  • target: 默认是 _self

    • 更改为 _blank 则目标页与本页面同时存在，默认则是目标页取代本页面

  • img: 视图 引入一张图片到本页面

    • src:source 的简写，表示图片的源
    • alt: 表示图片引入失败显示的内容
    • title: 鼠标悬浮显示的文本
    • 图片的大小如果不设置尺寸，则默认按照原图展示 这里我们使用 width 设置宽度，则默认会根据比例缩放图片

# css 引入方式

# 内嵌式

直接将 css 代码书写在 head 标签内，这种书写方式仅仅初步的将结构和样式解耦，但是解耦不彻底，复用性较差

# 外链式

引入一个外部独立的 css 文件到本页面，一个 html 页面不限制引入的 css 文件个数

rel: 表示引入的是一个 样式表 href: 表示独立 css 文件的路径

外链式的使用最为广泛，彻底的将结构和样式解耦，代码复用性极好

  • 链接中如果 href 设置为 #则点击时不会发送任何请求给目的地，浏览器会识别 #从而不发送同步请求

    如果书写其它字符，而没有资源，则浏览器发送请求报 404 错误，表示没有找到资源

# 行内式

直接将 css 代码书写在标签中，这种书写方式完全违背结构与样式解耦的原则

几乎没有重用性，但是由于其优先级极高，所以使用依然较多

当三种引入方式出现冲突时

​ 行内式 > 谁放在后面，会覆盖之前出现冲突的样式

# css 基本选择器

选择器分为 基本选择器和复合选择器两大类

# 基本选择器

  • 标记选择器

    ​ tagName{

    ​ 直接使用标签名作为选择元素的依据，这种书写方式极少单独使用，非常容易引起误操作

    ​ }

  • 类别选择器

    在标签中添加一个 class

    ​ .class{

    ​ 以。类名获取元素

    ​ }

    ​ 这个 class 属性不唯一

  • id 选择器

    在标签中添加 id

    ​ #id{

    ​ 以 #id 获取元素

    ​ }

    ​ id 属性上下文唯一，不能出现重复的 id 值

# 复合选择器

  • 交集选择器

    tagName.class{

    ​ }

    ​ tagName#id{

    ​ }

    ​ 由一个标签后面紧跟 class 或者 id, 则必须同时满足两个条件，才可以成功选取

  • 并集选择器

    一般在文档中将 选择器 省略 为 sel

    sel1,sel2,sel3,selN{

    ​ 由一个或者多个选择器组成，用逗号隔开，只要满足其中任意一个就可以成功选取

    ​ }

  • 后代选择器

    根据左祖先右后代的原则，用空格将具有特定层级关系的元素隔开

    可以精确拿取特定的子元素

  • 全选选择器

    *{

    ​ 拿取页面全部元素

    }

# span&&label

span 和 label 统称之为便签标签，这种标签没有换行，一行可以书写多个从左往右排列，多搭配 js 使用，单独使用没有任何效果

基本选择器的优先级问题

标记选择器 < 类别选择器 < id 选择器

id 选择器优先级最高，但是如果存在行内式，则一切以行内式为准

# css 元素类型

在 Css 中将元素分为很多种类型

主要分为以下两大类

# 行内元素 (inline 元素 内联元素)

eg: span label a input img …

​ 以上元素为行内元素，行内元素结尾没有换行，一行可以书写多个，从左往右排列，

​ 行内元素设置盒子模型大部分无效，设置对齐方式无效

​ 注意: input 和 img 元素较为特殊，它们又被称之为 行内块元素

​ 可以设置盒子模型参数

# 块元素 (block 元素)

eg: div p h1-h6 ul li table 布局元素

​ 页面上大部分的主要元素都属于块元素，这些块元素结尾自带换行，从上往下排列，

​ 设置盒子模型有效，设置对齐方式有效

设置元素的类型，元素原先的类型不是一成不变的，可以进行切换

​ display:inline; 设置元素为行内元素

​ display:block; 设置元素为块元素

display:none 隐藏元素

​ 浏览器其实为我们渲染了元素，但是最终不显示在页面上，我们可以理解元素上盖了一层蒙版

# css 盒子模型

标准文本流 (文档流 normal document flow)

​ 在页面中，我们书写的元素大多是块元素，块元素在不设置其宽度的前提，会与其父元素等宽，这些块元素由于结尾自带换行，所以从上往下排列，像水流一样自上而下，这种默认的状态称之为，标准文本流，在书写一个页面时，首先书写页面的 html 结构，当结构书写完毕呈现标准文本流时，开始书写 css 样式，当样式书写完毕时，开始书写 Javascript 脚本，html css js 是有先有后的书写顺序，切忌边写 html 边写 css

# 盒子模型

在 css 中不管元素是块元素还是行内元素，都将其看做一个一个的矩形的盒子，这些盒子根据元素的类型摆放在页面上 (一般是从上到下), 这些盒子存在三个重要参数

  • 盒子外面的空间 外边距 margin
  • 盒子边框 border
  • 盒子内部的控件 内边距 padding

在默认状态下 任何元素都不会显示边框 我们可以通过 border 属性显示元素的边框

  • border: 边框类型 边框粗细 边框颜色
  • 类型 solid 单实线 double 双实线 dotted 点状线

设置外边距

margin: 上 右 下 左；

margin: 上 (右左) 下；

margin:(上下) (右左);

margin:(上右下左);

# 浮动

是一种最简单的改变页面元素默认排列顺序 (标准文本流) 的 css 设置，块元素在设置浮动之后会随着浮动方向不断缩小，缩小到内部的内容为止，之后元素漂浮在页面上，原先的位置被后面的元素递增，继续保持标准文本流，如果之后的元素也设置浮动，则漂浮起来之后与第一个浮动的元素并排排列，如果设置了弹性盒子，则浮动全部失效

float:left 设置左浮动，从右往左缩小

float:right 设置右浮动，从左往右缩小

clear:left|right|both 防止元素收到标准文本流的影响

管道符：表示三个参数任选其一

list-style-type: none 去掉无序列表徽记

# 定位

定位是一种进阶的打破标准文本流的方式，元素可以彻底摆脱原先的排列方式，自由的摆放在页面的任何一个位置
定位分为四种我们主要介绍较难的两种

# 相对定位

元素根据其原先所在位置的左上角进行定位，top bottom left right 四个偏移量，可以设置元素的偏移位置，定位之后，元素依然保持原先的类型，原来的位置依然被占用

# 绝对定位

元素根据其定位过的距离最近的祖先元素的左上角进行定位，偏移量同相对定位，定位之后元素不再保持原先的类型，原来的位置被其它元素占用继续保持标注文本流，这个定位过的元素漂浮在页面中，如果元素没有定位过的祖先元素，则根据浏览器边框也就是 body 来进行定位，这种定位方式，在非全屏显示时，例如整体居中布局时极有可能造成错误

# 表格

table: 表示表格的开始
tr: 表示一行
td: 表示一列
横行竖列
以上三个元素缺一不可，表格最简不能再少于以上三个元素
table 嵌套 tr,tr 嵌套 td 顺序不可逆

caption: 表格标题，不是必须
thead: 用来嵌套表头，表头就是表格的第一行，不是必须，多搭配 css 使用
th: 效果同 td, 多使用在表头中，内部文本居中加粗，而 td 是左对齐不加粗
tbody: 用来嵌套表格主体，如果我们不书写，则自动添加，多搭配 css 使用

# Js 语法入门

Js 的基础语法由三部分组成，这三部分在早期是没有什么关联的，后来组合在一起组成 Js 的基本语法

  • ES 基本规范 (主要是 ES5 和 ES6 两个版本更新较大): 这部分是 Js 的语法主要组成部分，几乎没有浏览器差异性

    • Java 基本数据类型

      • byte char short int double long boolean float

    • Js 基本数据类型

      • string

        • “” ‘’ `` 都可以用来表示字符串
        • tostring () 转换为字符串

      • number

        • 表示数字，范围在 正负 2^53-1
        • let i = 100
        • number () 转换为数字
        • xxx-0 转换为数字 “10”-0 相当于 10
          • 转换为数字 +“10” 相当于 10
        • infinity 表示正无穷
        • -infinity 表示负无穷
        • NaN not a number 的简写 表示不是一个数字

      • boolean

        • true | false
        • 在 js 中 ‘’ null false 0 undefined 这些称之为 falsy 假值，除去这些值以外称之为 真值 truy
        • 在流程控制中 例如 if 后面 在 java 中必须书写布尔值，而在 js 中 if 后面 如果是真值就相当于 true, 假值就相当于 false, 所以我们在写 js 时要比 java 宽泛的多
        • if(arr.length)

      • null

        • 在 js 中使用 typeof 来判断数据的数据类型
        • 如果我们使用 typeof 测试 null 的数据类型，则返回 object, 这里是一个自 ES1 开始就存在的 bug, 迄今为止没有修改

      • undefined

        • 表示没有任何数据类型，在 js 中如果没有明确的数据类型，则统一称之为 undefined
